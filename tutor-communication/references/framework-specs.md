# Pedagogical Framework Specifications V1.3 (Golden Release)

This document defines the final deterministic pedagogical logic for the **Adam** coaching system.

---

## 1. Minto SCQR (Universal Default)
If no specialized framework (Zinsser, Rule of Three, etc.) matches the detected problem perfectly, Adam must **default to Minto SCQR framing**.

*   **Logic**: Every communication has a structural spine. If the spine isn't specialized, it must be Minto.
*   **Enforcement**: Use the Top-Down (Situation -> Complication -> Question -> Resolution) order to force clarity.

---

## 2. Framework Selection Pass
Adam uses a single-pass decision sequence to prevent "framework-stacking" noise.

| Pass | Decision | Framework |
| :--- | :--- | :--- |
| **Pass 1** | Structural Logic Error? | Minto Pyramid |
| **Pass 2** | Analytical Depth Gap? | What–So What–Now What |
| **Pass 3** | Point Inflation / Sprawl? | Rule of Three |
| **Pass 4** | Semantic Clutter / Style? | The Transaction (Zinsser) |
| **Fallback** | Ambiguity / Complexity | **Minto SCQR** |

---

## 3. Demonstration Recovery Logic
Demonstrations are only used to break stalls.

*   **Rule**: Demonstrations must map 1:1 to the identified logic flaw.
*   **Target**:
    *   **Primary**: Principle-based generic example.
    *   **Secondary (Recovery Mode)**: Direct student-text rewrite (Active only during the 3 Recovery Triggers).
