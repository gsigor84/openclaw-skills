## Term Definitions

---

### Skill
**Means:** A self-contained `SKILL.md` file (plus optional bundled resources) that instructs an OpenClaw agent to perform a specific, repeatable task when triggered.
**Does NOT mean:** A general capability, a Python script, or a prompt template.

---

### Niche
**Means:** The concrete, specific target domain for a skill being built (e.g. "fetch jazz playlists from Spotify"). Must be specific enough to probe with real tool calls.
**Does NOT mean:** A vague category (e.g. "music", "AI", "data").

---

### Observation
**Means:** The raw, unedited output of a single live tool call made during Phase 2. Recorded verbatim in `references/data/observations.md`.
**Does NOT mean:** A summary, paraphrase, or assumed result. If the tool wasn't called, it's not an observation.

---

### Experiment
**Means:** A deliberate variant test run during Phase 3 — a live tool call with a changed condition (e.g. with vs. without auth header). Recorded in `references/data/experiments.md`.
**Does NOT mean:** A thought experiment or hypothetical. Must be executed live.

---

### Rule
**Means:** An if-then statement derived from one or more observations or experiments, with an explicit source citation. Lives in `references/rules.md`.
**Does NOT mean:** A best practice, convention, or assumption. Rules without citations are deleted.

---

### Idol
**Means:** A cognitive bias (Tribe, Cave, Marketplace, or Theatre) that distorts reasoning during skill building. Named and logged in `references/idols-log.md`.
**Does NOT mean:** A mistake or bug. An idol is a bias pattern, not an error in code.

---

### Happy Path
**Means:** A tool call that returns the expected successful response (e.g. HTTP 200 with valid JSON shape).
**Does NOT mean:** The only path worth testing. Every skill must also probe failure paths.

---

### Edge Case
**Means:** A tool call that probes a boundary condition — empty input, auth wall (401/403), rate limit (429), null response, malformed data, or timeout.
**Does NOT mean:** An unlikely scenario to be skipped. At least one edge case is required to pass the Phase 2 gate.

---

### No-Signal
**Means:** The state where all Phase 3 experiments failed or returned no usable data. Declared explicitly in `references/data/experiments.md`. Limits Phase 4 to failure-mode rules only.
**Does NOT mean:** An incomplete session. No-signal is a valid, documented outcome.

---

### Traceability
**Means:** The ability to follow a chain from a SKILL.md behaviour → its rule in rules.md → its source observation or experiment. Verified in `references/traceability-matrix.md`.
**Does NOT mean:** Simply having files present. Traceability requires the links to be explicit and checkable.
